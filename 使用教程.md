## 1. 项目结构概览

```
DigitalGarden/
├── content/                    # ★ 所有内容都在这里
│   ├── posts/                  # 博客文章 (.md)
│   │   ├── p1.md
│   │   ├── p2.md
│   │   └── ...
│   ├── explore/                # 短动态 (.md)
│   │   └── ...
│   └── timeline/               # 时间轴条目 (.md)
│       └── ...
│
├── public/
│   ├── favicon.svg             # ★ 标签页图标（本地 SVG，直接编辑改样式）
│   └── assets/
│       └── images/             # 本地图片 (Markdown 中通过 /assets/images/... 引用)
│
├── src/
│   ├── config/
│   │   └── blog.config.ts      # ★ 全局配置——改这里就行
│   ├── app/                    # Next.js 页面路由
│   │   ├── layout.tsx          # 根布局 + metadata (SEO/OG/RSS/favicon)
│   │   ├── page.tsx            # 首页
│   │   ├── posts/              # /posts 文章列表 + /posts/[slug] 详情
│   │   ├── explore/            # /explore 动态流
│   │   ├── graph/              # /graph 知识图谱
│   │   └── timeline/           # /timeline 时间轴
│   ├── components/             # React 组件
│   ├── lib/                    # 数据解析逻辑
│   └── types/                  # TypeScript 类型定义
│
├── next.config.ts              # Next.js 配置 (图片域名白名单)
├── .env.example                # 环境变量模板
└── package.json
```

> **核心原则：** 新增或修改内容只需编辑 `content/` 目录下的 Markdown 文件。修改站点设置只需编辑 `src/config/blog.config.ts`。

## 2. 全局配置 blog.config.ts

**文件位置：** `src/config/blog.config.ts`

部署时只需修改这一个文件。所有组件和页面从此读取配置，项目中禁止硬编码站点信息。

### 2.1 站点身份

```ts
site: {
  name: "IStarry's Digital Garden",                     // 浏览器标签页标题、RSS、OG
  description: "A personal digital garden...",          // SEO meta description
  url: "https://istarry.top",                           // 生产域名 (含 https://)
  language: "zh-CN",                                    // <html lang> 属性
  locale: "en-US",                                      // 日期格式化 (toLocaleDateString)
}
```

### 2.2 作者信息

```ts
author: {
  name: "IStarry",
  bio: "A developer who writes about web, tools, and systems.",
  avatar: "https://img.istarry.top/images/head.jpg",    // 支持本地路径或远程 URL
}
```

### 2.3 品牌

```ts
branding: {
  favicon: "https://img.istarry.top/images/favicon.ico",  // OG metadata / RSS 使用
  logo: "https://img.istarry.top/images/favicon.ico",     // 站点 Logo
  ogImage: "/assets/images/og-default.png",               // 默认社交分享图
}
```

> **Favicon 机制：** 浏览器标签页图标由 `public/favicon.svg`（本地 SVG）提供，零延迟、不闪烁。`branding.favicon` 的远程 URL 用于 OG 分享、RSS 等场景。修改标签页图标只需编辑 `public/favicon.svg`，刷新即生效。

### 2.4 图片资源

```ts
assets: {
  localPath: "/assets/images",             // 本地图片目录
  cdnUrl: "https://img.istarry.top",       // 图床 CDN 前缀
  remoteDomains: [                         // 远程图片域名白名单
    "picsum.photos",
    "img.istarry.top",
  ],
}
```

> **⚠️ 新增图床域名时必须同步更新 `next.config.ts` → `images.remotePatterns`，否则 `cover` 封面图无法加载。**

### 2.5 Hero 区域

```ts
hero: {
  tagline: "Welcome to My Digital Garden 🌱",
  greeting: "Hi, I'm the developer...",
  highlight: "Systems, Security & Engineering Practice",
  description: "记录我的技术探索与工程实践...",
  buttons: {
    github: { label: "GitHub" },
    rss:    { label: "RSS Feed", href: "/rss.xml" },
  },

```

### 2.6 首页板块开关


### 2.11 站点分析 (Analytics)

支持**同时启用多个**统计服务，每个 provider 有独立的开关。

```ts
analytics: {
  enabled: true,                        // ★ 全局总开关
  providers: {
    umami: {
      enabled: false,                   // 是否启用 Umami
      id: "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
      scriptUrl: "https://cloud.umami.is/script.js",  // 自部署改为自己地址
    },
    google: {
      enabled: false,                   // 是否启用 Google Analytics
      measurementId: "G-XXXXXXXXXX",
    },
    la51: {
      enabled: false,                   // 是否启用 51la
      id: "3QY3fcmNgSjP54ai",          // 51la 网站 ID
      ck: "3QY3fcmNgSjP54ai",          // 51la ck 密钥（与 ID 不同）
    },
    baidu: {
      enabled: false,                   // 是否启用百度统计
      id: "7cff641d8826a80981ec7f2a89b5b77f",  // hm.js? 后面的 ID
    },
    clarity: {
      enabled: false,                   // 是否启用 Microsoft Clarity
      id: "xxxxxxxxx",                 // Clarity 项目 ID
    },
  },
}
```

> **使用方法：**
> 1. 将 `analytics.enabled` 设为 `true`
> 2. 将需要启用的 provider 的 `enabled` 设为 `true`
> 3. 填入对应服务的 ID/密钥
> 4. 支持同时启用多个，互不影响

**各服务 ID 获取地址：**

| 服务 | 获取地址 |
|------|---------|
| Umami | [cloud.umami.is](https://cloud.umami.is) → 添加网站 → 获取追踪 ID |
| Google Analytics | [analytics.google.com](https://analytics.google.com) → 数据流 → 衡量 ID (G-XXX) |
| 51la | [v6.51.la](https://v6.51.la) → 添加站点 → 获取 ID 和 ck 密钥 |
| 百度统计 | [tongji.baidu.com](https://tongji.baidu.com) → 添加站点 → 复制 hm.js 后的 token |
| Microsoft Clarity | [clarity.microsoft.com](https://clarity.microsoft.com) → 添加项目 → 获取项目 ID |

## 3. 文章 Posts

**文件位置：** `content/posts/*.md`

### 3.1 FrontMatter 完整字段

```yaml
---
title: "文章标题"                    # 必需
date: "2025-07-01"                   # 必需，YYYY-MM-DD
description: "文章摘要，用于列表和 SEO"  # 必需
tags:                                # 可选
  - cloud computing
  - Linux
category: "Cloud/Huawei"             # 可选，斜线分隔 → 分类层级
cover: "https://img.istarry.top/images/cover.png"  # 可选，封面图
relations:                           # 可选，知识图谱关系
  parent: "p2"                       # 父文章 slug
  related:                           # 相关文章 slug 列表
    - "p3"
  children:                          # 子文章 slug 列表
    - "p4"
---
正文（Markdown）...
```

### 3.2 字段规范

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `title` | string | **是** | 文章标题 |
| `date` | string | **是** | 发布日期 `YYYY-MM-DD` |
| `description` | string | **是** | 摘要，用于列表卡片 + SEO + RSS |
| `tags` | string[] | 否 | 标签，默认 `[]` |
| `category` | string | 否 | 斜线分隔路径，如 `"Linux/Security"` |
| `cover` | string | 否 | 封面图 URL（支持外部图床和本地路径） |
| `relations` | object | 否 | 知识图谱关系（见第 4 章） |

### 3.3 文件名规范

```
✅ p1.md                           ASCII 字符
✅ centos-deployment-guide.md      kebab-case
❌ 中文标题.md                     中文文件名 → URL slug 异常 → 404
❌ My Article Title.md             空格 → slug 含空格
```

> **文件名 = URL slug。** 标题用中文？写在 frontmatter `title` 字段里。

### 3.4 正文 Markdown 支持

- **标题：** `## H2` `### H3`（自动生成 TOC 导航锚点）
- **代码块：** ` ```ts ` 带语言标识（Shiki 双主题高亮，自动暗色模式切换）
- **行高亮：** ` ```ts {1,3-5} ` 高亮指定行
- **行号：** ` ```ts showLineNumbers `
- **图片：** `![描述](https://img.istarry.top/images/photo.png)` 或 `![描述](/assets/images/photo.png)`
- **表格：** GFM 表格语法
- **链接：** `[文字](url)`
- **粗体/斜体：** `**bold**` `*italic*`

## 4. 知识图谱 Graph

知识图谱由文章 frontmatter 中的 `relations` 字段自动构建。访问 `/graph` 查看交互式关系网络。

### 4.1 关系类型

| 关系 | 含义 | 建立方式 |
|------|------|---------|
| `parent` | 上级文章（前置知识） | 在子文章中声明 `parent: "父slug"` |
| `related` | 平级关联 | 任一方声明即可，系统自动双向补全 |
| `children` | 下级文章（进阶内容） | 在父文章中声明 `children: ["子slug"]` |

### 4.2 实际示例

`p2.md`（云计算概览——根节点）：
```yaml
category: "Cloud"
relations:
  children:
    - "p1"
    - "p3"
```

`p1.md`（华为云部署）：
```yaml
category: "Cloud/Huawei"
relations:
  parent: "p2"
  related:
    - "p3"
```

`p3.md`（CentOS 部署）：
```yaml
category: "Linux/Basics"
relations:
  parent: "p2"
  related:
    - "p1"
    - "linux-security-guide"
  children:
    - "p4"
    - "p6"
```

生成的图谱结构：

```
p2 (Cloud) ──────────────────────────── 根节点
├── p1 (Cloud/Huawei) ── related ── p3
└── p3 (Linux/Basics) ── related ── linux-security-guide
    ├── p4 (文件和用户管理)
    │   └── p5 (文件权限管理)
    └── p6 (进程管理)
```

### 4.3 category 分类层级

```yaml
category: "Linux/Security/Wazuh"
```

斜线分隔自动解析为 `["Linux", "Security", "Wazuh"]`，用于 `/graph` 页面的分类树。

## 7. 图片与资源引用

### 7.1 双轨兼容

```
方式 A（网络图床，推荐）          方式 B（本地路径，Git 追踪）
![图](https://img.istarry.top     ![图](/assets/images/photo.png)
      /images/photo.png)
      │                                   │
      线上加速                            本地预览
```

### 7.2 封面图

```yaml
# 外部图床（推荐）—— 不加引号也有效，但 URL 中包含特殊字符时建议加引号
cover: https://img.istarry.top/images/cover.png

# 本地资源
cover: /assets/images/posts/cover.png
```

> **注意：** `cover` 字段的值如果是简单 URL（纯英文/数字/短横线），YAML 的双引号可省略。但 URL 中含中文、空格或特殊符号时，必须加引号否则解析失败。

### 7.3 标签页图标 (Favicon)

标签页图标由 `public/favicon.svg` 提供，是本地 SVG 文件，不依赖任何外部 URL。

**修改方式：** 编辑 `public/favicon.svg`，改背景色（`fill="#1a1a2e"`）、文字色（`fill="#e94560"`）或显示文字（`IS`），保存后 `Ctrl+Shift+R` 硬刷新即可生效。

```svg
<!-- public/favicon.svg -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
  <rect width="32" height="32" rx="6" fill="#1a1a2e"/>
  <text x="16" y="22" text-anchor="middle" font-family="system-ui,sans-serif"
        font-size="18" font-weight="700" fill="#e94560">IS</text>
</svg>
```

> **注意：** 不要使用 `src/app/icon.tsx`（`next/og`）——在 Next.js 15.5 开发模式下会触发 Webpack 运行时错误。

### 7.4 ⚠️ 新增图床必须更新两处

1. `src/config/blog.config.ts` → `assets.remoteDomains` 数组
2. `next.config.ts` → `images.remotePatterns` 数组

**漏掉第二步会导致封面图（`next/image`）加载失败。**

### 7.5 正文图片不显示的排查

Markdown 正文中的 `![](url)` 图片不显示时，按顺序检查：

1. 直接在浏览器打开图片 URL——确认图片本身可访问
2. 检查图片 URL 是否有中文字符（图床通常返回 URL 编码后的地址，可直接使用）
3. 不回退到 `loading="lazy"`——项目已移除此属性，图片应正常即时加载

### 7.6 本地图片目录

```
public/
└── assets/
    └── images/
        ├── logo.png
        ├── avatar.png
        ├── og-default.png
        └── posts/
            └── example.png
```

## 8. 部署上线

### 8.1 环境变量

创建 `.env.local`（不提交到 Git）：

```bash
# 必需

# 评论系统 (可选)
NEXT_PUBLIC_GISCUS_REPO=istarry123/Digital-Garden
NEXT_PUBLIC_GISCUS_REPO_ID=R_kgDOGXXXXX
NEXT_PUBLIC_GISCUS_CATEGORY=General
```

## 9. 常见问题排查

### 封面图不显示

1. 确认域名在 `next.config.ts` → `images.remotePatterns` 中
2. 浏览器直接打开图片 URL——确认图片存在
3. 检查 `cover:` 后的 URL 格式（含特殊字符时需加引号）

### 正文图片不显示

1. 浏览器直接打开图片 URL 验证
2. 确保图片 URL 格式正确
3. 远程图床是否允许跨域访问

### 标签页图标不更新

1. 确认 `public/favicon.svg` 存在
2. `Ctrl+Shift+R` 硬刷新（浏览器对 favicon 缓存极其顽固）
3. 或用隐私/无痕窗口测试

### 文章 404

1. 文件名是否含中文字符（→ URL slug 异常）
2. 文件名是否含空格
3. frontmatter 中 `date` 字段格式是否为 `YYYY-MM-DD`

### npm run dev 报错

```bash
rm -rf .next          # 清除构建缓存
npm run dev           # 重新启动
```
