## 1. 项目结构概览

```
DigitalGarden/
├── content/                    # ★ 所有内容都在这里
│   ├── posts/                  # 博客文章 (.md)
│   │   ├── hello-world.md
│   │   ├── linux-security-guide.md
│   │   └── ...
│   ├── explore/                # 短动态 (.md)
│   │   ├── 2026-07-10-excited-nextjs.md
│   │   └── ...
│   └── timeline/               # 时间轴条目 (.md)
│       ├── 2016-started-programming.md
│       └── ...
│
├── public/
│   └── assets/
│       └── images/             # 本地图片 (Markdown 中通过 /assets/images/... 引用)
│
├── src/
│   ├── config/
│   │   └── blog.config.ts      # ★ 全局配置——改这里就行
│   ├── app/                    # Next.js 页面路由
│   ├── components/             # React 组件
│   ├── lib/                    # 数据解析逻辑
│   └── types/                  # TypeScript 类型定义
│
├── .env.example                # 环境变量模板
└── package.json
```

> **核心原则：** 新增或修改内容只需编辑 `content/` 目录下的 Markdown 文件。修改站点设置只需编辑 `src/config/blog.config.ts`。

## 2. 全局配置 blog.config.ts

**文件位置：** `src/config/blog.config.ts`

部署时只需修改这一个文件。关键配置项：

### 2.1 站点身份

```ts
site: {
  name: "Digital Garden",                              // 浏览器标签页标题、RSS、OG
  description: "A personal digital garden...",         // SEO meta description
  url: "https://digitalgarden.vercel.app",             // 生产域名
  language: "zh-CN",                                   // <html lang> 属性
  locale: "en-US",                                     // 日期格式化 (toLocaleDateString)
}
```

### 2.2 作者信息

```ts
author: {
  name: "Digital Garden",
  bio: "A developer who writes about web, tools, and systems.",
  avatar: "/images/avatar.png",                        // 头像放在 public/images/ 下
}
```

### 2.3 品牌

```ts
branding: {
  favicon: "/favicon.ico",                             // 浏览器标签页图标
  logo: "/images/logo.png",                            // 站点 Logo
  ogImage: "/images/og-default.png",                   // 默认社交分享图
}
```

### 2.4 Hero 区域 (首页个人介绍)

```ts
hero: {
  tagline: "Welcome to my corner of the internet",
  greeting: "Hi, I&apos;m a developer",
  highlight: "web, tools &amp; systems",               // 高亮文字 (& 用 &amp;)
  description: "This is my digital garden...",          // 个人描述
  buttons: {
    github: { label: "GitHub" },
    rss:    { label: "RSS Feed", href: "/rss.xml" },
  },
}
```

> **注意：** `highlight` 中的 `&` 符号必须写成 `&amp;`。

### 2.5 首页板块开关

```ts
homepage: {
  showLatestArticles: false,   // 设为 true 恢复首页文章卡片
  showLatestExplore: false,    // 设为 true 恢复首页动态流
}
```

### 2.6 社交链接

Hero 按钮中的 GitHub 链接，和 Navbar 下拉菜单的社交图标分别配置：

```ts
social: {
  github: "https://github.com",       // Hero 按钮使用
  twitter: "@digitalgarden",          // Twitter Card metadata
},

socialLinks: [                         // Navbar 下拉菜单
  { name: "GitHub",  url: "https://github.com",           icon: "github" },
  { name: "Twitter", url: "https://twitter.com",          icon: "twitter" },
  { name: "Email",   url: "mailto:hello@example.com",     icon: "email" },
],
```

**新增社交平台：** 在 `socialLinks` 数组中添加对象，然后在 `src/components/layout/social-dropdown.tsx` 的 `ICON_MAP` 中添加对应 SVG。

### 2.7 子页面元数据

```ts
pages: {
  posts:    { title: "Articles",        heading: "Articles",        description: "...", emptyText: "No articles yet.", },
  explore:  { title: "Explore",         heading: "Explore",         description: "...", emptyText: "Nothing here yet.", },
  graph:    { title: "Knowledge Graph", description: "..." },
  timeline: { title: "Timeline",        description: "..." },
}
```

### 2.8 评论系统 (Giscus)

需要在 GitHub 仓库启用 Discussions，然后在 [giscus.app](https://giscus.app) 获取 ID：

```bash
# .env.local (不提交到 Git)
NEXT_PUBLIC_GISCUS_REPO=your-username/your-repo
NEXT_PUBLIC_GISCUS_REPO_ID=R_kgDOGXXXXX
NEXT_PUBLIC_GISCUS_CATEGORY=General
NEXT_PUBLIC_GISCUS_CATEGORY_ID=DIC_kwDOGXXXXX
```

未配置时评论区域不渲染。

### 2.9 导航栏

```ts
navigation: {
  articles: { label: "Articles", title: "Articles", iconKey: "articles" },
  explore:  { label: "Explore",  title: "Explore",  iconKey: "explore" },
  rss:      { label: "RSS Feed", title: "RSS Feed", iconKey: "rss" },
  graph:    { label: "Knowledge Graph", title: "Knowledge Graph", iconKey: "graph" },
  timeline: { label: "Timeline", title: "Timeline", iconKey: "timeline" },
}
```

## 3. 文章 Posts

**文件位置：** `content/posts/*.md`

### 3.1 FrontMatter 完整字段

```yaml
---
title: "文章标题"                    # 必需
date: "2026-07-01"                   # 必需，ISO 日期格式 YYYY-MM-DD
description: "文章摘要，用于列表和 SEO"  # 必需
tags:                                # 可选，标签列表
  - Next.js
  - React
  - TypeScript
category: "Web/Next.js"              # 可选，斜线分隔分类路径
cover: "https://picsum.photos/1200/630"  # 可选，封面图 URL
relations:                           # 可选，知识图谱关系
  parent: "linux-security-guide"     # 父文章 slug（文件名去掉 .md）
  related:                           # 相关文章 slug 列表
    - "building-digital-garden"
  children:                          # 子文章 slug 列表
    - "wazuh-agent-setup"
---

正文（Markdown 格式）...
```

### 3.2 字段规范

| 字段 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `title` | string | **是** | — | 文章标题 |
| `date` | string | **是** | — | 发布日期 ISO 格式 `YYYY-MM-DD` |
| `description` | string | **是** | — | 文章摘要，用于列表卡片、SEO、RSS |
| `tags` | string[] | 否 | `[]` | 标签，用于标签聚合 + 相关文章推荐 |
| `category` | string | 否 | `""` | 斜线分隔路径，如 `"Linux/Security"` → 分类树 |
| `cover` | string | 否 | 无 | 封面图 URL（支持外部链接如 picsum.photos） |
| `relations` | object | 否 | `{}` | 知识图谱关系配置（见下方详解） |

### 3.3 文件名规范

```
✅ hello-world.md                  ASCII字符，kebab-case
✅ centos-deployment-guide.md      英文连字符
❌ 中文标题.md                     中文文件名导致 URL slug 异常
❌ My Article Title.md             空格导致 slug 含空格
```

> **文件名 = URL slug。** 标题、描述等身份信息全部通过 frontmatter 声明。

### 3.4 正文 Markdown 支持

- **标题：** `## H2` `### H3`（自动生成 TOC 导航锚点）
- **代码块：** ` ```ts ` 带语言标识（Shiki 双主题高亮）
- **行号：** ` ```ts {1,3-5} ` 高亮指定行
- **图片：** `![alt](/images/photo.png)`（图片放 `public/images/`，自动懒加载 + 圆角阴影）
- **表格：** GFM 表格语法
- **链接：** `[text](url)`
- **粗体/斜体：** `**bold**` `*italic*`

## 4. 知识图谱 Graph

知识图谱由文章 frontmatter 中的 `relations` 字段自动构建，**无需手动创建**。

访问 `/graph` 查看交互式关系网络。

### 4.1 关系类型

```
                      ┌──────────────────┐
                      │  Linux 安全基础指南 │  ← 根节点 (无 parent)
                      └────────┬─────────┘
                               │ parent_child (实线)
                               ▼
          ┌────────────────────────────────────┐
          │       CentOS 系统部署和基本操作      │
          │  parent: "linux-security-guide"    │
          │  related: ["building-digital-garden"] │ ← 蓝色虚线
          └────────────────────────────────────┘
```

### 4.2 关系配置规则

```yaml
relations:
  parent: "父文章slug"         # 单值——建立上下级层级关系
  related:                      # 数组——建立平级关联
    - "文章A的slug"
    - "文章B的slug"
  children:                     # 数组——声明子文章
    - "子文章slug"
```

| 关系 | 含义 | 建立方式 |
|------|------|---------|
| `parent` | 当前文章的上级（知识基础/前置阅读） | 在子文章中声明 `parent: "父slug"` |
| `related` | 平级关联（话题相关） | 任意一方声明即可，系统自动双向补全 |
| `children` | 当前文章的下级（进阶/细化） | 在父文章中声明 `children: ["子slug"]` |

> **自动双向补全：** A 声明 `related: [B]`，系统自动将 A 加入 B 的 `related` 列表。A 声明 `parent: B`，系统自动将 A 加入 B 的 `children` 列表。

### 4.3 category 分类层级

```yaml
category: "Linux/Security/Wazuh"
```

斜线分隔的路径自动解析为 `["Linux", "Security", "Wazuh"]`，在 `/graph` 页面中按层级展示分类标签。

## 5. 个人时间轴 Timeline

**文件位置：** `content/timeline/*.md`

### 5.1 FrontMatter 字段

```yaml
---
year: 2026                          # 必需，年份（用于分组排序）
title: "企业安全体系建设"             # 必需，条目标题
description: "主导部署 Wazuh SIEM..."  # 必需，简短描述
tags:                               # 可选，技术标签
  - Wazuh
  - Seafile
  - 安全架构
---

可选正文（Markdown）——用于详细描述、列表、链接。
每条记录都会出现在时间轴的对应年份下方。
```

### 5.2 字段规范

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `year` | number | **是** | 四位年份，如 `2026`。用于分组和排序 |
| `title` | string | **是** | 条目标题 |
| `description` | string | **是** | 简短摘要（显示在卡片顶部） |
| `tags` | string[] | 否 | 技术标签列表 |
| 正文 | markdown | 否 | 可选详细内容（支持完整 Markdown） |

### 5.3 命名建议

```
content/timeline/
├── 2016-started-programming.md     ✅ 年份-短描述
├── 2019-first-security-job.md
├── 2022-built-homelab.md
├── 2024-cloud-migration.md
└── 2026-enterprise-security.md
```

年份降序自动排列（最新年份在最上面）。

## 7. 部署上线

### 7.1 环境变量

```bash
# 必需
NEXT_PUBLIC_SITE_URL=https://your-domain.vercel.app

# 评论系统 (可选)
NEXT_PUBLIC_GISCUS_REPO=your-username/your-repo
NEXT_PUBLIC_GISCUS_REPO_ID=R_kgDOGXXXXX
NEXT_PUBLIC_GISCUS_CATEGORY=General
NEXT_PUBLIC_GISCUS_CATEGORY_ID=DIC_kwDOGXXXXX

# 站点分析 (可选)
NEXT_PUBLIC_UMAMI_ID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

### 7.2 Vercel 部署步骤

1. **推送代码到 GitHub**
   ```bash
   git add .
   git commit -m "Ready for deploy"
   git push origin main
   ```

2. **导入 Vercel**
   访问 [vercel.com/new](https://vercel.com/new)，选择你的 GitHub 仓库。Vercel 自动识别 Next.js 项目。

3. **配置环境变量**
   在 Vercel 项目 → Settings → Environment Variables 中添加上述变量。

4. **部署**
   点击 Deploy。每次 `git push` 自动触发重新部署。

### 7.3 内容更新工作流

```
1. 在 content/ 下新建 .md 文件或编辑已有文件
2. git add . && git commit -m "New post: xxx"
3. git push origin main
4. Vercel 自动检测 push → 触发构建 → 所有静态页面更新
```

> 无需服务器、无需数据库、无需手动操作。所有内容在构建时编译为静态 HTML，通过 Vercel 全球 CDN 分发。

### 7.4 本地开发

```bash
npm install          # 安装依赖
npm run dev          # 启动开发服务器 → http://localhost:3000
npm run build        # 生产构建验证
npm run lint         # ESLint 检查
```

### 图片与资源引用（双轨兼容）

所有图片支持两种引用方式，本地编辑和线上部署均能正常显示。

**目录结构：**

```
public/
└── assets/
    └── images/          ← 所有本地图片放这里
        ├── logo.png
        ├── avatar.png
        ├── og-default.png
        └── posts/       ← 文章配图（可选子目录）
            └── example.png
```

**Markdown 正文中引用：**

```markdown
<!-- 网络图床（优先，适用于线上加速） -->
![图片描述](https://your-cdn.com/assets/images/photo.png)

<!-- 本地路径（Git 追踪，本地编辑器预览） -->
![图片描述](/assets/images/photo.png)
```

**启用 CDN 加速（可选）：**

在 `.env.local` 中设置：

```bash
NEXT_PUBLIC_IMAGE_CDN=https://cdn.example.com
```

设置后，所有 `/assets/images/...` 路径自动替换为 CDN URL。留空则从本地 `/public/` 提供。

**封面图：**

```yaml
# 外部图床
cover: "https://picsum.photos/seed/xxx/1200/630"

# 本地资源
cover: "/assets/images/posts/cover.png"
```

**自动优化：** 所有图片自动获得懒加载、圆角、阴影、暗黑模式适配（通过 `rehype-image-optimizer` 插件）。
