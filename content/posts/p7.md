## 为什么选择数字花园？

数字花园不同于博客。博客是按时间顺序排列的、打磨过后的文章流。而数字花园是不断演化的想法集合——笔记、随笔和半成品思想，它们会随着时间生长。它是一个**公开思考**的地方，而不仅仅是发布成品。

我选择使用 Next.js 15 构建它的原因有三：

1. **React 服务端组件**——默认向客户端发送零 JavaScript
2. **文件系统路由**——`app/` 目录直接映射到 URL
3. **Vercel**——一次 `git push` 即可完成部署

## 项目搭建

脚手架非常精简。这是核心配置：

```ts
// next.config.ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  reactStrictMode: true,
};

export default nextConfig;
```

配合 Tailwind CSS v4，CSS 设置只需两行：

```css
@import "tailwindcss";
@variant dark (&:where(.dark, .dark *));
```

`@variant` 指令将深色模式从基于媒体查询切换为基于类的方式，这是 `next-themes` 正常工作的必要条件。

## 内容管道

我选择将 Markdown 文件存放在 `content/` 目录下作为内容数据库。管道如下：

- **gray-matter** 解析 frontmatter（标题、日期、标签、描述）
- **remark** + **rehype** 在构建时将 Markdown 转换为 HTML
- **Tailwind Typography** 通过 `prose` 类为渲染后的 HTML 添加样式

这意味着内容渲染完全不需要客户端 JavaScript——所有内容都在构建时预渲染完成。

## 主题系统

主题系统使用 `next-themes`，提供三种模式：

| 模式 | 行为             |
| :--- | :--------------- |
| 浅色 | 始终使用浅色     |
| 深色 | 始终使用深色     |
| 系统 | 跟随操作系统偏好 |

导航栏中的 `<ThemeToggle />` 组件允许访客切换模式。偏好会保存在 `localStorage` 中，因此刷新页面不会丢失。

## 关键心得

> 适合个人网站的最佳工具，就是你自己乐于维护的那个。

Next.js 15 取得了很好的平衡——足够强大以应对复杂应用，又足够简洁以胜任博客。而且借助 React 服务端组件，开箱即用的性能表现非常出色。
