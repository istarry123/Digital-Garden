# 第一篇 基础环境

## 第1章 Seafile 13.0 概述

### 1.1 版本定位

Seafile 13.0 是一个里程碑版本，将 Seafile 从单纯的同步分享工具转变为更智能、更协作、AI 增强的文件平台。

### 1.2 13.0 重大新特性

**1. 元数据服务器（Metadata Server）**
全新的元数据服务器组件，允许为每个文件定义结构化属性——如项目阶段、负责人、状态、到期日期或任何自定义字段。元数据可被搜索、过滤、分组和排序。

**2. 多视图显示**
支持表格视图、看板视图、画廊视图、地图视图、卡片视图和统计视图。

**3. 层级标签管理**
支持多级标签层级结构，可表达标签之间的从属关系。

**4. 实时 UI 更新**
启用通知服务器后，当其他用户添加或删除文件时，Web 界面实时更新。

**5. 专用缩略图服务器**
新增缩略图服务器组件，提升缩略图生成性能，支持视频缩略图。

**6. Seafile AI**
AI 可提取文件详情和文本、生成描述和标签、检测人脸等。

**7. SeaSearch 1.0**
生产级全文搜索。

**8. SeaDoc 2.0**
块编辑器、AI 辅助、白板（Excalidraw）。

**9. Wiki**
基于块的编辑、评论、Ask AI、sdoc/Markdown 导入导出。

### 1.3 系统要求

| 部署场景         | CPU          | 内存 | 磁盘  |
| :--------------- | :----------- | :--- | :---- |
| 社区版（Docker） | 2核（>2GHz） | 2GB+ | 50GB+ |
| 专业版（Docker） | 4核（>2GHz） | 4GB+ | 50GB+ |

生产环境建议：4核 CPU + 8GB 内存 + 100GB+ 磁盘。

### 1.4 架构变化

Seafile 13.0 引入了以下架构变化：

- 数据库和 Memcache 配置移至 `.env` 文件，推荐使用环境变量配置数据库和缓存
- Redis 推荐作为 Memcache 服务器
- S3 配置可通过环境变量完成，大幅简化
- ElasticSearch 拥有独立的 yml 文件
- Docker 镜像中的 Nginx 不再从映射卷读取配置

### 1.5 破坏性变更

| 变更项               | 说明                                                         |
| :------------------- | :----------------------------------------------------------- |
| 二进制部署（社区版） | **不再支持**                                                 |
| WebDAV LDAP 登录     | 因安全原因不再支持，LDAP 用户需在个人资料页生成 WebDAV Token |
| 旧文件标签           | 旧标签功能不再可用，界面提供升级通知迁移到新标签系统         |

> **从 12.0 升级的用户**：必须先迁移到 Docker 部署，再升级到 13.0。

## 第2章 系统初始化

### 2.1 Ubuntu 24.04 基础配置



```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install -y curl wget gnupg ca-certificates \
    net-tools htop iotop sysstat vim git unzip \
    build-essential pkg-config

# 设置时区
sudo timedatectl set-timezone Asia/Shanghai

# 启用 NTP 同步
sudo timedatectl set-ntp true
sudo systemctl enable --now systemd-timesyncd
```



### 2.2 安装 Docker 和 Docker Compose



```bash
# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 将当前用户加入 docker 组
sudo usermod -aG docker $USER

# 安装 Docker Compose（独立版本）
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证
docker --version
docker-compose --version
```



### 2.3 防火墙配置（UFW）



```bash
sudo ufw --force enable
sudo ufw default deny incoming
sudo ufw default allow outgoing

sudo ufw allow 22/tcp comment 'SSH'
sudo ufw allow 80/tcp comment 'HTTP'
sudo ufw allow 443/tcp comment 'HTTPS'

# 如需 OnlyOffice/Collabora
sudo ufw allow 9980/tcp comment 'Collabora Online'

sudo ufw status numbered
```



# 第二篇 Seafile 13.0 Docker 部署

## 第3章 下载与配置

### 3.1 创建目录并下载配置文件



```bash
# 创建 Seafile 目录
sudo mkdir -p /opt/seafile
cd /opt/seafile

# 下载 Docker Compose 配置文件[reference:29]
sudo wget -O .env https://manual.seafile.com/13.0/repo/docker/ce/env
sudo wget https://manual.seafile.com/13.0/repo/docker/ce/seafile-server.yml
sudo wget https://manual.seafile.com/13.0/repo/docker/seadoc.yml
sudo wget https://manual.seafile.com/13.0/repo/docker/caddy.yml
```



### 3.2 配置 .env 文件

编辑 `.env` 文件，配置以下关键参数：



```bash
sudo nano .env
```



**必须配置的参数**：

| 变量                               | 说明                            | 示例                    |
| :--------------------------------- | :------------------------------ | :---------------------- |
| `SEAFILE_VOLUME`                   | Seafile 数据卷目录              | `/opt/seafile-data`     |
| `SEAFILE_MYSQL_VOLUME`             | MySQL 数据卷目录                | `/opt/seafile-mysql/db` |
| `SEAFILE_CADDY_VOLUME`             | Caddy 证书存储目录              | `/opt/seafile-caddy`    |
| `INIT_SEAFILE_MYSQL_ROOT_PASSWORD` | MySQL root 密码（首次部署必需） | `your_strong_password`  |
| `SEAFILE_MYSQL_DB_PASSWORD`        | Seafile 数据库用户密码          | `your_db_password`      |
| `SEAFILE_SERVER_HOSTNAME`          | 服务器域名                      | `cloud.your-domain.com` |

**13.0 新增配置**：



```bash
# 存储后端类型（13.0 新增）[reference:31][reference:32]
# 留空使用 seafile.conf 中的旧配置方式
SEAF_SERVER_STORAGE_TYPE=

# 缓存配置（13.0 推荐使用 Redis）[reference:33]
CACHE_PROVIDER=redis
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=

# 或使用 Memcached
# CACHE_PROVIDER=memcached
# MEMCACHED_HOST=memcached
# MEMCACHED_PORT=11211
```



### 3.3 启动 Seafile



```bash
cd /opt/seafile

# 首次启动（会自动拉取镜像并初始化）
sudo docker-compose -f seafile-server.yml up -d

# 查看运行状态
sudo docker-compose -f seafile-server.yml ps

# 查看日志
sudo docker-compose -f seafile-server.yml logs -f
```



### 3.4 创建管理员账户



```bash
# 进入 Seafile 容器
sudo docker exec -it seafile /bin/bash

# 在容器内创建管理员
cd /opt/seafile/seafile-server-latest
./seahub.sh python-env manage.py createsuperuser
```



### 3.5 验证部署

访问 `https://your-domain.com`，使用管理员账户登录。

如果使用 Let's Encrypt 自动证书（通过 Caddy），确保域名 DNS 解析正确。

### 4.3 配置 Nginx/Caddy 代理

如果使用 Caddy（默认），SeaDoc 会自动配置路由。如需手动配置 Nginx：



```nginx
location /seadoc/ {
    proxy_pass http://127.0.0.1:8080/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```



## 第5章 部署缩略图服务器（13.0 新增）

### 5.2 启用缩略图服务器

缩略图服务器默认随 Seafile 主容器启动。如需独立配置：



```bash
# 检查缩略图服务状态
sudo docker exec -it seafile ps aux | grep thumbnail
```

在 `seafile.conf` 中可配置缩略图参数：



```ini
[thumbnail]
# 最大缩略图大小（像素）
max_size = 1024
# 缩略图质量（1-100）
quality = 85
# 支持的视频格式
video_formats = mp4,avi,mov,wmv,flv,mkv
```

## 第7章 部署通知服务器（13.0 新增）

### 7.1 概述

通知服务器支持实时 UI 更新——当其他用户在同一资料库同一文件夹中添加或删除文件时，界面自动更新，无需手动刷新。

### 7.2 启用通知服务

通知服务器随 Seafile 主容器自动启动。确认配置：



```bash
# 检查通知服务
sudo docker exec -it seafile ps aux | grep notification
```



在 `seafile.conf` 中可配置：



```ini
[notification]
enabled = true
# 通知轮询间隔（秒）
polling_interval = 5
```



## 第8章 部署 Seafile AI（13.0 新增）

### 8.1 概述

Seafile AI 为日常文件工作带来实用自动化：

- 提取文件详情和文本

- 自动生成标签

### 8.2 启用 Seafile AI

Seafile AI 需要元数据服务启用后使用。

1. 确保元数据服务已启用
2. 在系统管理 → AI 设置中配置 AI 服务
3. 配置 AI 模型 API（如 OpenAI API 或本地模型）

### 8.3 AI 配置示例



```ini
[ai]

provider = openai
# API Key

# 模型

# 启用功能
enable_extract = true
enable_description = true
enable_tagging = true
enable_face_detection = true
```

## 第9章 Collabora Online（LibreOffice 在线编辑）

### 9.1 概述

Seafile 支持集成 Collabora Online 进行在线预览和编辑 Office 文档。

### 9.2 部署 Collabora Online（Docker）



```bash
# 拉取 Collabora 镜像
docker pull collabora/code

# 启动容器
docker run -t -d -p 127.0.0.1:9980:9980 \
    -e "aliasgroup1=https://your-domain.com:443" \
    -e "username=admin" \
    -e "password=your_password" \
    --name collabora --restart always \
    collabora/code
```



### 9.3 配置 Seafile

在 Seafile 容器的 `seahub_settings.py` 中：



```python
# 启用 Collabora Online
OFFICE_SERVER_TYPE = 'CollaboraOffice'
OFFICE_SERVER_URL = 'https://collabora.your-domain.com'
```



重启 Seafile：



```bash
sudo docker restart seafile
```



### 9.4 Nginx 代理配置



```nginx
location ^~ /browser {
    proxy_pass https://127.0.0.1:9980;
    proxy_set_header Host $http_host;
}

location ~ ^/cool/(.*)/ws$ {
    proxy_pass https://127.0.0.1:9980;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "Upgrade";
    proxy_read_timeout 36000s;
}
```

## 第14章 备份与恢复

### 14.1 备份内容

Seafile Docker 部署需要备份：

1. **数据卷**：`/opt/seafile-data/`
2. **MySQL 数据卷**：`/opt/seafile-mysql/`
3. **配置文件**：`/opt/seafile/.env` 和 `*.yml`

### 14.2 备份脚本



```bash
#!/bin/bash
# /opt/scripts/backup-seafile.sh

BACKUP_DIR="/backup/seafile"
DATE=$(date +%Y%m%d_%H%M%S)
SEAFILE_DIR="/opt/seafile"

mkdir -p $BACKUP_DIR

# 停止服务
cd $SEAFILE_DIR
sudo docker-compose -f seafile-server.yml down

# 备份数据卷
sudo tar -czf $BACKUP_DIR/seafile-data-$DATE.tar.gz /opt/seafile-data/
sudo tar -czf $BACKUP_DIR/seafile-mysql-$DATE.tar.gz /opt/seafile-mysql/

# 备份配置
sudo tar -czf $BACKUP_DIR/seafile-config-$DATE.tar.gz \
    $SEAFILE_DIR/.env \
    $SEAFILE_DIR/*.yml

# 启动服务
sudo docker-compose -f seafile-server.yml up -d

# 保留最近30天备份
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete
```



### 14.3 恢复步骤



```bash
# 1. 停止并移除现有容器
cd /opt/seafile
sudo docker-compose -f seafile-server.yml down

# 2. 恢复数据卷
sudo tar -xzf /backup/seafile/seafile-data-20260101.tar.gz -C /
sudo tar -xzf /backup/seafile/seafile-mysql-20260101.tar.gz -C /

# 3. 恢复配置
sudo tar -xzf /backup/seafile/seafile-config-20260101.tar.gz -C /

# 4. 启动服务
sudo docker-compose -f seafile-server.yml up -d
```



## 第15章 升级

### 15.1 从 12.0 升级到 13.0

> **重要**：Seafile 13.0 社区版不再支持二进制部署。如果使用 12.0 二进制部署，**必须先迁移到 Docker 部署**，再升级到 13.0。

**迁移路径**：

1. 在 12.0 版本先将二进制部署迁移到 Docker 部署
2. 再按照 Docker 升级流程升级到 13.0

### 15.2 Docker 升级步骤（13.0.x 小版本）



```bash
cd /opt/seafile

# 1. 停止服务
sudo docker-compose -f seafile-server.yml down

# 2. 备份当前配置
cp .env .env.bak
cp seafile-server.yml seafile-server.yml.bak

# 3. 更新 .env 中的镜像版本
# SEAFILE_IMAGE=seafileltd/seafile-mc:13.0-latest

# 4. 如从 12 升级，下载新的 compose 文件[reference:44]
wget -O seafile-server.yml https://manual.seafile.com/13.0/repo/docker/ce/seafile-server.yml

# 5. 更新 .env 配置（13.0 新增）[reference:45]
# 添加缓存配置
echo "CACHE_PROVIDER=redis" >> .env
echo "REDIS_HOST=redis" >> .env
echo "REDIS_PORT=6379" >> .env

# 6. 启动服务
sudo docker-compose -f seafile-server.yml up -d

# 7. 运行升级脚本（容器内）
sudo docker exec -it seafile /bin/bash
cd /opt/seafile/seafile-server-latest
./upgrade/upgrade_12_13.sh
```



### 15.3 升级注意事项

升级前需执行：

1. **清理数据库表**：运行升级脚本前清理

2. **安装新系统库和 Python 库**：

   

   ```bash
   apt-get install -y python3 python3-dev python3-setuptools python3-pip \
       python3-ldap python3-rados libmysqlclient-dev ldap-utils \
       libldap2-dev python3.12-venv default-libmysqlclient-dev
   
   pip3 install --timeout=3600 boto3 oss2 twilio configparser pytz \
       sqlalchemy==2.0.* pymysql==1.1.* jinja2 django-pylibmc pylibmc redis
   ```

## 第17章 性能优化

### 17.1 使用 Redis 缓存（13.0 推荐）

从 Seafile 13.0 开始，Redis 被推荐作为主要缓存服务器，以支持新功能。

在 `.env` 中配置：



```bash
CACHE_PROVIDER=redis
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=your_redis_password
```



### 17.2 数据库优化

在 `seafile.conf` 中：



```ini
[database]
# 连接池大小
pool_size = 20
# 最大连接数
max_connections = 100
```



### 17.3 缩略图性能（13.0 新增）

```ini
[thumbnail]
# 并发生成线程数

```



# 第五篇 故障排查

## 第18章 常见错误与解决方案

### E001：Docker 容器无法启动

**现象**：



```text
Error response from daemon: Conflict. The container name "/seafile" is already in use
```



**解决方案**：



```bash
sudo docker-compose -f /opt/seafile/seafile-server.yml down
sudo docker-compose -f /opt/seafile/seafile-server.yml up -d
```



### E002：数据库连接失败

**现象**：



```text
Can't connect to MySQL server on 'db'
```



**原因**：MySQL 容器未就绪或密码配置错误

**解决方案**：



```bash
# 检查 MySQL 容器状态
sudo docker ps | grep mysql

# 查看 MySQL 日志
sudo docker logs seafile-mysql

# 检查 .env 中的数据库密码
```



### E003：Let's Encrypt 证书申请失败

**现象**：



```text
Failed to obtain certificate
```



**原因**：域名 DNS 解析未指向服务器，或 80/443 端口被占用

**解决方案**：



```bash
# 确认域名解析
dig your-domain.com

# 确认端口开放
sudo netstat -tlnp | grep -E "80|443"
```



### E004：SeaDoc 无法加载

**现象**：



```text
Failed to load SeaDoc
```



**解决方案**：



```bash
# 检查 SeaDoc 容器状态
sudo docker-compose -f /opt/seafile/seadoc.yml ps

# 查看 SeaDoc 日志
sudo docker-compose -f /opt/seafile/seadoc.yml logs
```



### E005：缩略图不显示（13.0）

**现象**：图片或视频缩略图无法生成

**解决方案**：



```bash
# 检查缩略图服务器状态
sudo docker exec -it seafile ps aux | grep thumbnail

# 查看缩略图日志
sudo docker exec -it seafile cat /opt/seafile/logs/thumbnail-server.log
```



### E006：迁移旧标签失败（13.0）

**现象**：升级后旧标签不可用

**原因**：13.0 不再支持旧标签系统

**解决方案**：Web 界面会显示升级通知，点击迁移数据到新标签系统。

### E007：WebDAV 认证失败（13.0）

**现象**：



```text
WebDAV authentication failed
```



**原因**：13.0 不再支持 LDAP 账户直接登录 WebDAV

**解决方案**：LDAP 用户在个人资料页生成 WebDAV Token，使用 Token 认证。

### E008：磁盘空间不足

**解决方案**：



```bash
# 运行 GC
sudo docker exec -it seafile /opt/seafile/seafile-server-latest/seaf-gc.sh

# 清理 Docker 无用数据
sudo docker system prune -f
```



### E009：元数据服务不可用（13.0）

**现象**：无法启用元数据功能

**解决方案**：



```bash
# 检查元数据服务
sudo docker exec -it seafile ps aux | grep metadata

# 检查 seafile.conf 配置
sudo docker exec -it seafile cat /opt/seafile/conf/seafile.conf | grep metadata
```
