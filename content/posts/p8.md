## Tailwind v4 有哪些变化？

Tailwind CSS v4 对深色模式做了一个细微但重要的改动。在 v3 中，你需要在 `tailwind.config.ts` 中配置深色模式：

```ts
// Tailwind v3
export default {
  darkMode: "class",
  // ...
};
```

而在 v4 中，**根本没有配置文件**。相反，深色模式默认使用 `prefers-color-scheme` 媒体查询。若要切换为基于类（class）的模式，你需要在 CSS 中覆盖 `dark` 变体：

css

```
@import "tailwindcss";
@variant dark (&:where(.dark, .dark *));
```



## 为什么要用基于类的模式？

基于媒体查询的深色模式（`prefers-color-scheme: dark`）对于只跟随操作系统设置来说工作良好。但对于一个带有手动主题切换开关的博客，你需要基于类的模式。

`next-themes` 库通过在 `<html>` 元素上添加 `class="dark"` 来工作：

html

```
<html class="dark" lang="zh-CN">
```



然后 Tailwind 的 `dark:` 前缀会匹配该 class 内部的元素：

html

```
<div class="bg-white dark:bg-neutral-950">
  这个 div 会根据主题自适应
</div>
```



## 避免水合不匹配

在 Next.js 中实现深色模式最棘手的部分是避免**错误主题闪烁**。解决方案如下：

1. 在 `<html>` 上添加 `suppressHydrationWarning`
2. 使用 `next-themes`，它会在 React 水合之前注入一个内联 `<script>`
3. 在切换组件中，使用 `mounted` 状态模式：

tsx

```
const [mounted, setMounted] = useState(false);
useEffect(() => setMounted(true), []);

if (!mounted) {
  return <div className="h-9 w-9" />; // 占位
}
```



这样可以确保服务端和客户端在首次渲染时输出完全相同的 HTML。

## `@variant` 语法详解

`@variant dark (&:where(.dark, .dark *))` 这一行使用了 `:where()` 伪类。这很重要，因为 `:where()` 具有**零特异性**——它不会意外覆盖其他样式。

对比一下：

| 选择器                                     | 特异性  |
| :----------------------------------------- | :------ |
| `.dark .dark\:text-white`                  | `0,2,0` |
| `:where(.dark, .dark *) .dark\:text-white` | `0,1,0` |

使用 `:where()` 可以让 CSS 层叠更可预测。
